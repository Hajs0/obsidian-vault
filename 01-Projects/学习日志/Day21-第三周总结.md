## 四、关键学习收获

### 1. Express 中间件模式
```
请求 → 中间件1 → 中间件2 → 路由处理 → 响应
              ↓
          错误处理中间件
```
- 中间件是 Express 的核心概念
- 错误处理中间件有 4 个参数 (err, req, res, next)
- 中间件顺序很重要

### 2. JWT 认证流程
```
登录 → 验证密码 → 生成 Token → 返回客户端
                                   ↓
请求 → 携带 Token → 验证 Token → 获取用户信息
```
- 无状态认证，不需要服务端存储
- Token 有过期时间
- 需要保护好 JWT_SECRET

### 3. Prisma ORM 优势
- 类型安全的数据库操作
- 自动生成的客户端
- 迁移管理方便
- 支持多种数据库

### 4. 测试的重要性
- 单元测试：测试独立函数
- 集成测试：测试 API 端点
- 测试覆盖率：确保代码质量

## 五、遇到的问题和解决方案

### 问题 1: TypeScript 类型错误
**现象：** Prisma 查询结果类型不正确  
**解决：** 使用 Prisma 生成的类型，或者用 `as` 类型断言

### 问题 2: JWT Token 过期处理
**现象：** Token 过期后前端没有正确处理  
**解决：** 拦截 401 响应，自动刷新 Token 或跳转登录

### 问题 3: 数据库连接泄漏
**现象：** 测试结束后进程不退出  
**解决：** 在应用关闭时断开 Prisma 连接

```typescript
process.on('SIGTERM', async () => {
  await prisma.$disconnect();
  process.exit(0);
});
```

### 问题 4: CORS 跨域问题
**现象：** 前端无法访问后端 API  
**解决：** 配置 CORS 中间件，允许指定来源

```typescript
app.use(cors({
  origin: process.env.CORS_ORIGIN || 'http://localhost:3001',
  credentials: true
}));
```

### 问题 5: Docker 构建慢
**现象：** 每次构建都要重新安装依赖  
**解决：** 利用 Docker 缓存层，先复制 package.json
